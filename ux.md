| 心理効果／原則                                          | 内容の要点                                                                       | UX 上の意味・応用                                                                  |
| ------------------------------------------------------- | -------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **美的ユーザビリティ効果 (Aesthetic-Usability Effect)** | 美しいデザインのものは、美しくないものに比べ「使いやすい／良い」と感じられやすい | 見た目のデザインに気を配ることで、実際の使い勝手以上に好印象を与えられる           |
| **アンカー効果 (Anchor Effect)**                        | ユーは最初に見た情報（アンカー）で後の判断を左右されやすい                       | 最初に提示する価格や情報でユーザーの判断基準をコントロールできる                   |
| **バナー・ブラインドネス (Banner Blindness)**           | 広告バナーなどを、人は無意識に無視しがち                                         | 広告や重要な案内を“バナー風”で配置すると、見落とされやすい                         |
| **認知負荷 (Cognitive Load)**                           | 情報を処理する負荷の大きさ。多すぎると使いづらく感じる                           | 情報や操作を整理／簡潔にすることで、UX を改善できる                                |
| **確証バイアス (Confirmation Bias)**                    | 自分の既存の信念に合う情報を優先し、反するものを無視しやすい                     | ユーザーが持つ先入観を理解した上で、デザインやメッセージを提示する必要がある       |
| **好奇心ギャップ (Curiosity Gap)**                      | 情報の「穴」を見せることで、知りたいという欲求を刺激する                         | コンテンツや情報設計で興味を引き、行動（スクロール、クリック）を促す               |
| **決断疲れ (Decision Fatigue)**                         | 多くの決断を繰り返すと、判断力が低下する                                         | 選択肢を減らしたり、デフォルト設定を活用したりして UX を楽にする                   |
| **おとり効果 (Decoy Effect)**                           | あえて“おとり”の選択肢を入れることで、他の選択肢を魅力的に見せる                 | プランや価格設計において、ユーザーの選択を操作できる                               |
| **デフォルト効果 (Default Bias)**                       | 提示された初期設定（デフォルト）を変えずに使い続ける傾向                         | 初期設定を慎重に設計することで、多くのユーザーの行動を誘導できる                   |
| **“0.4 秒の壁” (ドハティの閾値 / Doherty Threshold)**   | ユーザーが 0.4 秒以上待たされると、関心を失いやすい                              | 読み込みや反応速度をできるだけ速くする必要がある                                   |
| **共感ギャップ (Empathy Gap)**                          | デザイナーとユーザーの感情・立場のズレ                                           | 実際のユーザー視点を想像／観察して、設計を行うべき                                 |
| **授かり効果 (Endowment Effect)**                       | 所有しているものを、過大に価値あると感じがち                                     | ユーザーに「所有」感を持たせることで、プロダクトの価値を高められる                 |
| **期待バイアス (Expectation Bias)**                     | 事前の期待が体験評価に影響する                                                   | ユーザーの期待値を管理し、過剰な期待を避ける工夫が必要                             |
| **親近性バイアス (Familiarity Bias)**                   | 慣れ親しんだデザイン・機能に好感を抱きやすい                                     | 過度な革新より、「馴染み」を残すデザインも重要                                     |
| **段階的要請 (Foot in the Door Effect)**                | 小さな要求から始めると、大きな要求も受け入れやすい                               | サインアップ → 購入 など段階的に誘導する設計に有用                                 |
| **フレーミング効果 (Framing)**                          | 情報の提示の仕方（言葉や枠）で判断が変わる                                       | 表現や文言、順序を工夫して、ユーザーの受け取り方を制御できる                       |
| **ゲーミフィケーション (Gamification)**                 | ゲーム的要素を取り入れて、モチベーションを高める                                 | 継続利用や行動促進に向けたデザイン手法として有効                                   |
| **目標勾配効果 (Goal Gradient Effect)**                 | 目標に近づくほどモチベーションが高まる                                           | 進捗表示やステップ形式で、達成感を与える設計が効果的                               |
| **ハロー効果 (Halo Effect)**                            | ある良い要素の印象が、全体の評価を高める                                         | ブランドイメージや最初の体験の質が、プロダクト全体の評価に影響する                 |
| **観察効果 (Hawthorne Effect)**                         | “見られている”と感じることで、人は行動を変える                                   | ユーザーに評価されている・注目されているという感覚を与える設計も強みになる         |
| **意図的な壁 (Intentional Friction)**                   | あえて操作や手数を残すことで、価値を感じさせる                                   | 全てを“スムーズ”にせず、適度なステップを残すことで価値感を作る                     |
| **労働の錯覚 (Labor Illusion)**                         | 手間がかかっているように見せることで、サービスの価値を高める                     | バックグラウンド処理や演出で「手間」に見せる使い方もある                           |
| **損失回避 (Loss Aversion)**                            | 利得よりも“損を避ける”ことを重視する傾向                                         | 期限や“失う可能性”を示すことで、人を行動に促すことができる                         |
| **ナッジ (Nudge)**                                      | 強制せず、軽く背中を押すような誘導                                               | ユーザーの自発性を尊重しつつ、望ましい行動に導く設計に使える                       |
| **ピーク・エンドの法則 (Peak-End Rule)**                | 体験を評価する際、最も強い時と終わりの印象が重要                                 | UI の“山場”や“終わり”を丁寧に設計することが大事                                    |
| **プライミング (Priming)**                              | 先に与えた情報や刺激が、その後の行動に影響する                                   | 初めの印象や情報で、その後の行動を誘導しやすい                                     |
| **段階的開示 (Progressive Disclosure)**                 | 必要なときに必要な情報だけを出す設計                                             | 情報量を抑えつつ、必要に応じて詳細を見せることで負荷を減らす                       |
| **ピグマリオン効果 (Pygmalion Effect)**                 | 期待した通りの結果を人が引き出す傾向                                             | ユーザーにポジティブな期待を持たせることで、体験の満足度を上げやすい               |
| **誘導抵抗 (Reactance)**                                | 制限されたと感じると反発したくなる心理                                           | 過度な誘導・制限は逆効果になりうるので注意が必要                                   |
| **反応型オンボーディング (Reactive Onboarding)**        | ユーザーの行動や状態に応じて、導入プロセスを調整する方法                         | ユーザーごとに最適な案内や操作を提示できる柔軟な設計                               |
| **希少性効果 (Scarcity)**                               | 入手困難・限定などで、価値を高く感じやすい心理                                   | 限定性や希少性を演出することで、購入や行動を促せる                                 |
| **選択的注意 (Selective Attention)**                    | 必要な情報に注意を向け、それ以外を無視する傾向                                   | 重要な要素を強調し、それ以外を抑えることで情報過多を防ぐ                           |
| **系列位置効果 (Serial Position Effect)**               | リストなどで、最初と最後の項目が覚えられやすい                                   | メニューや選択肢の並び順を工夫することで、印象や記憶を操作できる                   |
| **スキューモーフィズム (Skeuomorphism)**                | 現実のものを模したデザインで理解を促進する手法                                   | 新しい UI でも、慣れ親しんだ見た目を使うと学習コストが下がる                       |
| **社会的証明 (Social Proof)**                           | 他者の評価・行動を見せることで、自分もそれに従いやすくなる                       | レビューや利用数、事例の提示で信頼や行動促進につながる                             |
| **サンクコスト効果 (Sunk Cost Effect)**                 | 過去の投資（時間・お金）を無駄にしたくない心理                                   | 継続利用やステップ型サービスで、“途中までやったら…”という心理を刺激できる          |
| **誘惑の結びつけ (Temptation Bundling)**                | 好きなことと、やりたくないことを一緒にすることで行動を促す                       | 難しい操作やタスクに、楽しい要素を組み込むことでユーザーへの心理的負担を軽減できる |
| **ユーザー歓喜効果 (User Delight)**                     | 使っていて驚きや喜びを感じるような瞬間をデザインすること                         | 単なる機能性ではなく、“感情的に嬉しい体験”を重要視する                             |
| **変動型報酬 (Variable Reward)**                        | 報酬が一定でなく、予測できないことで、行動のモチベーションが続きやすい           | 継続利用、習慣化、リテンションに効果的な設計手法                                   |
